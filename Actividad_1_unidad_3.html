En el presente caso de esta base de datos, la serie de tiempo no es estacionaria, vemos que la variabilidad no es muy estable pues esta se incrementa al principio y luego disminuye considerablemente y presenta, en general, una tendencia positiva. Se aprecia tambin un cambio en la media con el tiempo por la tendencia positiva de los datos. Por ende, podemos considerar primeramente realizar un ajuste quiz logartmico, es decir utilizar ajuste tipo Box-Cox. Sin embargo, la serie presenta datos negativos y no podremos calcular \(\lambda\) o \(ln X_{t}\) para algunos valores de la variable aleatoria. En este caso, es mejor considerar el mtodo de diferencias o datos diferenciados \(Y_{t}= \nabla X_{t}\) o \[\nabla X_{t}= X_{t} - X_{t-1}\] Con el software R esto es posible con la funcin diff() y el resultado al graficar los nuevos datos es:

x1<- Dat[,2]
x <- ts(x1)
Dat_Dif<-diff(x,1)
ts.plot(Dat_Dif, main = "Datos Ajustados por Diferencial (1)", xlab="Tiempo", ylab="Datos")

Ntese ahora se tiene un comportamiento estacionario, de hecho, los datos se distribuyen alrededor de su media a lo largo del tiempo y dicha media es:

En el modelo ajustado, los correlogramas muestran muchos picos no nulos. Vemos en ambos que se hacen completamente nulos despus de Lag 4 aproximadamente. Podemos entonces proponer algn modelo \(MA(4)\) o \(AR(4)\) que modele los datos. Para ver cual se ajusta mejor, utilizamos en R el Criterio Akaike \((AIC)\); el modelo con un coeficiente menor AIC ser el ms adecuado, esto es:

AIC(arima(Dat_Dif,order = c(0,0,4)),arima(Dat_Dif,order = c(4,0,0)))

Vase como un \(AR(4)\) parece ser mejor, sin embargo un \(MA(4)\) no est lejos segn el criterio empleado. Podemos entonces proponer un \(AR(4)\) Ajustando el modelo a los datos se tiene: Primero para saber qu modelo \(AR(p)\) es mejor para nuestros datos, usamos la funcin \(ar()\), esto nos dir tanto el orden como los coeficientes del modelo mediante sus \(MLE\). El resultado es:

Vemos como nuestra prediccin a partir de los correlogramas y el criterio \(AIC\) concuerdan con lo arrojado por \(ar()\) y vemos como incluso \(\sigma^{2}= 1\) con lo que el modelo para los datos sera: \[ X_{t} = 0.1026X_{t-1}+0.0367X_{t-2}+0.0846X_{t-3}+0.1178X_{t-4}+\epsilon_{t}\] Donde \(\epsilon_{t}\) es ruido blanco donde \(\mu=0\) y \(\sigma^{2}= 1\). Ahora, podemos hacer una simulacin con el modelo obtenido para comparar los datos generados con los originales y ver si el comportamiento es similar. La grfica de la simulacin es:

phi1<-0.1026 
phi2<-0.036
phi3<-0.0846
phi4<-0.1178
n<-300
  
mburn <- 300

m <- n + mburn

y <- {1:m}
tiempo <- {1:n}

# generar ruido blanco
eps <- rnorm(m)

y[1] <- eps[1]
y[2] <- eps[2]
y[3] <- eps[3]
y[4] <- eps[4]

# generar proceso AR(1) en vec y[]
for(i in {5:m}){
  
  y[i] <- phi1*y[i-1] +phi2*y[i-2]+phi3*y[i-3]+phi4*y[i-4] + eps[i]}

# tirar las primeras mburn observaciones
y <- y[(mburn+1):m]
  dat<- data.frame(tiempo,y)
  
  # graficar la serie de tiempo
  bp<- ggplot(data=dat, aes(x=tiempo, y=y, group=1)) +

En conclusin, hemos hecho un ajuste de los datos que tenamos originalmente de manera que la serie de tiempo fuera estacionaria. Para ello utilizamos datos diferenciados \(Y_{t}= \nabla X_{t}\) con \(\nabla X_{t}= X_{t} - X_{t-1}\) despus graficamos y calculamos los correlogramas de los nuevos datos. De ellos nos servimos para poder estimar el tipo de modelo tipo \(ARMA(p,q)\) que mejor se ajustara a la nueva serie estacionaria utilizando el software R. Concluimos de nuestro anlisis que el modelo que representa nuestros datos es un \(AR(4)\) y hemos hecho una simulacin con ste donde nos damos cuenta que los datos generados y sus correlogramas presentan un comportamiento muy similar, lo cual nos dice, que efectivamente hemos hecho una buena estimacin a partir de la transformacin que hicimos con los datos originales.
